## 📐 2. KEY DEFINITIONS (One-Liner Each)

1. **Cyber Security** — Protection of computers, networks, data from cyber attacks.
2. **Cyber Crime** — Criminal activity using computers/internet.
3. **CIA Triad** — Confidentiality, Integrity, Availability.
4. **Hacking** — Unauthorized access to computer systems.
5. **Phishing** — Fraudulent email tricking users into revealing data.
6. **Ransomware** — Malware that encrypts files demanding ransom.
7. **DDoS** — Flooding server from multiple sources to deny service.
8. **SQL Injection** — Inserting malicious SQL into web inputs.
9. **Cyber Terrorism** — Politically-motivated attacks for terror.
10. **Cyber Warfare** — State-vs-state cyber attacks.
11. **Digital Signature** — Cryptographic equivalent of handwritten signature.
12. **PKI** — Public Key Infrastructure for digital trust.
13. **Certifying Authority (CA)** — Issues digital signature certificates.
14. **CCA** — Controller of Certifying Authorities (regulatory body).
15. **Cyber Forensics** — Scientific process of digital evidence handling.
16. **IPR** — Intellectual Property Rights (Copyright, Patent, Trademark).
17. **Cybersquatting** — Registering trademark domain names for resale.
18. **E-Governance** — Use of ICT for government service delivery.
19. **Authentication** — Verifying who the user is.
20. **Authorization** — Verifying what the user can do.

## 🆚 4. CRUCIAL DIFFERENCE TABLES

### 4.1 Active vs Passive Attacks
| Feature | Active | Passive |
|---------|--------|---------|
| Action | Modifies data | Only observes |
| Detection | Easier | Hard |
| Prevention | Hard | Easier (encryption) |
| Examples | DoS, replay, masquerade | Eavesdropping |

### 4.2 Authentication vs Authorization
| Feature | Authentication | Authorization |
|---------|----------------|---------------|
| Purpose | Identify user | Determine access |
| Order | First | After authentication |
| Methods | Password, OTP, biometric | Roles, ACLs |
| Question | Who are you? | What can you do? |

### 4.3 Symmetric vs Asymmetric Crypto
| Feature | Symmetric | Asymmetric |
|---------|-----------|------------|
| Keys | One shared | Public + Private |
| Speed | Fast | Slow |
| Algorithms | DES, AES, 3DES | RSA, ECC, DSA |
| Use | Bulk encryption | Key exchange, signatures |

### 4.4 Copyright vs Patent
| Feature | Copyright | Patent |
|---------|-----------|--------|
| Protects | Expression of idea | Inventions |
| Duration | Lifetime + 60 yrs | 20 yrs |
| Registration | Not mandatory | Mandatory |
| Indian Law | Copyright Act 1957 | Patents Act 1970 |

### 4.5 Cyber Crime vs Cyber Terrorism vs Cyber Warfare
| Feature | Cyber Crime | Cyber Terrorism | Cyber Warfare |
|---------|-------------|-----------------|----------------|
| Motive | Financial/personal | Political/terror | National (state) |
| Target | Individual/biz | Civilians/infra | Nation |
| Section | 65, 66, 67, 72 | 66F | – |
| Punishment | Years + fine | Life imprisonment | – |

### 4.6 Residual Risk vs Risk Appetite
| Feature | Residual Risk | Risk Appetite |
|---------|---------------|---------------|
| Definition | Risk after controls | Risk willing to take |
| Nature | Calculated | Decided |
| Direction | Bottom-up | Top-down |

### 4.7 GDPR vs DPDP Act
| Feature | GDPR (EU) | DPDP Act (India) |
|---------|-----------|-------------------|
| Year | 2018 | 2023 |
| Penalty | €20M / 4% turnover | Up to ₹250 crore |
| Authority | DPAs | Data Protection Board |
| Right to be forgotten | Yes | Yes |

### 4.8 Hacker Types
| Type | Motive | Legal? |
|------|--------|--------|
| White Hat | Ethical, hired | Legal |
| Black Hat | Malicious | Illegal |
| Grey Hat | Mixed | Often illegal |
| Script Kiddie | Inexperienced | Illegal |
| Hacktivist | Ideological | Illegal |

### 4.9 Forward Proxy vs Reverse Proxy (relevant to firewalls)
| Forward Proxy | Reverse Proxy |
|---------------|----------------|
| For clients | For servers |
| Hide internal network | Hide backend servers |
| Used in offices/schools | Used by website operators |

### 4.10 Symmetric Cryptography Algorithms

|-----------|----------|------------|
| DES | 56 bits | 64 bits |

## 🔑 6. KEY FORMULAS & FRAMEWORKS

### CIA Triad
```
   Confidentiality
       /\
      /  \
     /CIA \
    /Triad \
   /________\
Integrity    Availability
```

### Risk Formula
> **Residual Risk = Inherent Risk – Risk mitigated by controls**

### RSA Algorithm
1. Choose primes p, q
2. n = p × q
3. φ(n) = (p-1)(q-1)
4. Choose e: gcd(e, φ(n)) = 1
5. d: e × d ≡ 1 mod φ(n)
6. **Encrypt:** C = M^e mod n
7. **Decrypt:** M = C^d mod n

### Hash + Signature Process
```
Sender: Doc → Hash → Encrypt(Private) → Signature
Receiver: Decrypt(Public) → Hash A vs Hash B
If equal → Authentic + Unaltered ✓
```

### Digital Forensics Lifecycle
```
Identify → Preserve → Collect → Examine → Analyze → Present
```

### Defense in Depth Layers
```
Physical → Perimeter → Network → Endpoint → App → Data → IAM → Ops → DR
```

## 🎯 11. EXAM DAY STRATEGY

### Time Management:
- **2-mark questions:** ~3-5 mins each
- **5-mark questions:** ~10-12 mins each
- **10-mark questions:** ~20-25 mins each

### Order of Attempt:
1. Start with **2-mark short-answers** (build confidence, quick marks)
2. Then **5-mark questions** you know best
3. Finally **10-mark theory** with diagrams (most time-consuming)

### Power Phrases (boost marks):
- "Section X of IT Act 2000 deals with..."
- "Punishment is imprisonment up to X years and/or fine up to ₹X lakh."
- "In the landmark case of *XYZ vs UoI*, the Supreme Court held that..."
- "The CIA Triad consists of..."
- "Recent developments include the DPDP Act 2023..."

### Diagram Strategy:
- **CIA Triad** → easy 1-mark visual
- **Digital Signature** → flow diagram
- **Forensics Lifecycle** → 6-stage flowchart
- **DDoS Attack** → attacker → botnet → target
